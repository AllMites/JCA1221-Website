Implementation Approaches

Two viable paths for the icon-to-video carousel

A: Native <video> + Custom Hook

React state machine with native video elements, useRef for playback control, useEffect timers for auto-rotation. CSS box-shadow glow.

Pros

  • Zero new dependencies
  • Matches existing patterns (useEffect + refs)
  • Simple video seek/play/pause API
  • Easy to swap placeholder → real files later

Cons

  • Manual video orchestration (play, pause, currentTime reset)
  • Cross-browser video quirks (autoplay policies)

B: framer-motion AnimatePresence

AnimatePresence wraps video transitions with crossfade. Leverages existing framer-motion dependency. Videos fade between each other.

Pros

  • Smooth crossfade transitions
  • Already in the project
  • AnimatePresence handles mount/unmount lifecycle

Cons

  • Two video elements mounted simultaneously during crossfade (memory)
  • Harder to coordinate play state across fading elements
  • Overkill — the "hold last frame" is already a natural pause

Recommendation: Approach A

Single <video> element whose src swaps on icon change. The "hold last frame" behavior means no crossfade needed — the transition feels natural. Matches existing code patterns. No new deps.

Both approaches use the same state machine underneath — only the video transition strategy differs.
